# 《时空语航员》Cocos 独立 App 重构设计规格

**文档状态：** Target（已锁定为本会话确认后的设计真值）  
**适用目标版本：** Cocos 独立 App v2  
**当前代码基线：** Web/PWA MVP，仓库 `main` 分支，审计截至 2026-07-15  
**目标平台：** iOS、Android 原生独立 App  
**目标引擎：** Cocos Creator 3.8.8 + TypeScript  
**首发形态：** 家庭自用或小范围测试，完全离线  
**目标用户：** 小学 1–6 年级，按低年级与高年级自适应  

### 会话决策锁定（2026-07-15）

| 决策点 | 锁定结论 |
| --- | --- |
| 首要目标 | 学习效果 + 长期留存 |

| 执行节奏 | 按 master plan 顺序推进，不再逐节产品闸门确认 |

## 1. 已确认的产品决策

1. 不做微信小游戏，构建 iOS/Android 原生独立 App。
2. 不在现有 Web UI 上渐进改造，使用 Cocos Creator 全量迁移。
3. 允许推翻薄弱系统设计，但必须服务“学习效果 + 长期留存”；推图/突袭/多模态答题闭环不得丢失。
4. 核心体验采用“电影化指令 RPG + 少量 Roguelite 二选一强化”（方案甲增强形态）。
5. 答题采用全屏专注弹层；答题时暂停战斗演出，答完后播放无遮挡的全屏攻击。
6. 首发覆盖小学 1–6 年级，交互和策略深度按年龄自适应。
7. 首发不建设账号、云服务、广告、内购、在线活动或第三方分析。
8. 美术和音频以 AI、开源素材和开发者加工为主。
9. 不使用 Spine 或其他付费骨骼编辑工具。
10. 武器采用 5 个少而精的机制流派；宠物采用 3 个少而精的三阶段伙伴。

## 2. 当前 Web MVP 真值与主要偏差

### 2.1 当前实际能力

当前项目是无构建链、无后端、无测试的纯前端 PWA：

- 原生 JavaScript + HTML + CSS；
- `localStorage` v4 多孩子存档；
- 双教材：沪教牛津 2024 与沪教版英语口语交际；
- 覆盖 1A–6B；
- 推图战斗和艾宾浩斯复习突袭；
- 选择、听力、阅读、拼写、口语题；
- 5 件武器、3 只宠物、最多双宠出战；
- 11 档军衔、10 项成就、学情摘要；
- WebAudio 合成音效、浏览器 TTS、DOM 粒子；
- 存档导入导出。

### 2.2 文档与代码冲突

现有 README 和 GDD 不能继续作为开发真值：

- 文档写 4 级秒级复习，代码为 5 级 20 分钟至 7 天；
- 文档写 Word/Dialogue/Reading 三形态，代码为听/读/拼/说四 Boss；
- 文档将口语评测写成未来能力，代码已经实现；
- 文档写宇航服、尾迹和星际花园，实际主要实现的是武器和宠物；
- 花园只有字段、样式和不可达成成就，没有可玩 UI；
- GDD 受众为 3–6 年级，代码和教材覆盖 1–6 年级；
- GDD 的 3A Unit 1《Hello》已不符合 2024 教材数据；
- 怪兽资源 ID 与运行时 Boss ID 不一致，四阶段显示同一回退怪兽；
- `player.suit` 实际保存武器 ID，飞船皮肤与武器语义混乱；
- `assets/svg` 外部资源未被运行时引用；
- 无关卡解锁、无真正花园、无后台威胁扣血或偷水晶；
- 零自动化测试、零埋点、零后端。

### 2.3 对重构的约束

- 旧代码是业务规则和教材数据的参考，不作为新架构模板。
- 教材内容、孩子档案、学习记录和已获得物品应尽可能迁移。
- 不迁移现有全局对象、字符串拼接 UI、巨型数据常量和 DOM 特效实现。
- 新系统必须把学习真值、游戏规则和画面表现分离。

## 3. 产品定位

### 3.1 一句话定义

《时空语航员》是一款把教材同步英语练习转化为电影化舰队指令战斗的离线儿童学习 RPG。

### 3.2 设计目标

优先级从高到低：

1. 到期知识点的首次回忆正确率持续提升；
2. 孩子愿意主动完成短时、持续的复习；
3. 听、说、读、写题型都获得足够清晰的交互空间；
4. 战斗、武器和宠物拥有可感知的机制差异与演出差异；
5. 家长能理解孩子学了什么、掌握到什么程度；
6. 个人开发者能够持续生产和维护内容。

### 3.3 非目标

- 不做实时 PvP、排行榜、公会或社交聊天；
- 不做抽卡、体力付费、广告、限时活动或连续签到惩罚；
- 不以手速、闪避或复杂操作决定学习成败；
- 不让装备、宠物或等级修改正确答案、掌握判定或复习时间；
- 不在首发建设服务器和云账号。

## 4. 分龄体验

### 4.1 1–3 年级

- 无强制答题倒计时；
- 默认 2×2 大按钮，最小触控高度 64 dp；
- 长句选项改为 1×4 纵向布局；
- 拼写使用字母块，不弹系统键盘；
- 战术强化只展示两个图标化选项；
- 宠物技能自动释放；
- Boss 攻击提前预告，失败保护更强；
- 每局 6–8 题，目标时长 5–8 分钟。

### 4.2 4–6 年级

- 可在家长设置中启用温和倒计时；
- 拼写可切换字母块或完整键盘；
- 开放 2 个永久武器模块与更多局内强化；
- Boss 阶段增加弱点、打断和资源取舍；
- 每局 8–12 题，目标时长 6–10 分钟。

### 4.3 共同规则

- 口语识别失败不能判定为学习错误；
- 所有动效都可降低或关闭；
- 所有关键状态必须同时通过图形、文字和声音中的至少两种方式表达；
- 颜色不能作为唯一状态区分。

## 5. 核心循环

### 5.1 局外循环

1. 进入基地；
2. 查看主线、到期突袭和三个短任务；
3. 配置一件主武器、两个武器模块、一只先锋宠和一只支援宠；
4. 选择任务并进入战斗；
5. 获得知识掌握进度、航员经验、合金、星晶、武器进度和宠物羁绊；
6. 升级、进化或调整构筑；
7. 根据新的复习到期时间再次出发。

### 5.2 单局循环

1. **入场：** Boss 登场，显示本局知识目标和弱点提示；
2. **弱点侦测：** 学习引擎选择当前题目；
3. **全屏下令：** 战斗画面暂停或降为静态背景，进入完整题型 UI；
4. **指令结算：** 计算首次答对、纠错通过、跳过或设备失败；
5. **战斗演出：** 播放武器攻击、Boss 受击和宠物连携；
6. **阶段检查：** 每 3 题提供一次二选一强化，或触发 Boss 转阶段；
7. **终局：** 清空核心阶段后播放终结技；
8. **结算：** 分别展示学习结果与游戏奖励。

### 5.3 单局长度

- 主线：8–12 题，6–10 分钟；
- 到期复习：4–8 题，3–5 分钟；
- 弱项专项：6–10 题，5–8 分钟；
- 任何模式不得通过重复简单题无限产生有效奖励。

## 6. 学习系统

### 6.1 知识点模型

每个词汇、句型或对话单元使用稳定 `contentId`，至少记录：

- 教材、年级、学期、单元；
- 类型：词汇、句型、对话；
- 英文、中文、音频引用；
- 可用题型；
- 干扰项标签；
- 学习状态、稳定度、难度、到期时间；
- 首次答对次数、纠错次数、跳过次数、设备失败次数。

### 6.2 掌握阶段

前端显示五阶段：

1. 新接触；
2. 初识；
3. 短期掌握；
4. 稳固；
5. 长期掌握。

内部调度不再只使用固定层级冷却，而使用离线可运行的 FSRS 式稳定度和难度模型。游戏中的“怪兽威胁等级”是调度结果的表现层，不是另一套学习数据。

### 6.3 题目选择

一局题目按以下优先级采样：

1. 已到期复习；
2. 当前单元弱项；
3. 当前任务指定新内容；
4. 为避免连续同题型而加入的平衡项。

约束：

- 同一知识点不能连续出现；
- 答错知识点延后 2–3 个题位再次出现；
- 一局内首次答对后的重复不再产生完整经验；
- 干扰项必须与题目同类型、同难度范围且不含重复正确答案；
- 低年级避免四个语义过近的干扰项。

### 6.4 答题结果

- 首次答对：指令质量 1.0，完整学习奖励；
- 纠错后答对：指令质量 0.6，部分奖励，不增加动量；
- 跳过：指令质量 0.2，不判学习错误；
- 语音识别或设备失败：指令质量 0.2，记录设备失败，不增加错误次数；
- 真正答错：增加错误次数，降低稳定度并安排重现。

### 6.5 口语

- 主判定使用目标文本与识别文本的标准化相似度；
- 识别不可用时允许跟读录音并自我确认，但不能获得首次满额奖励；
- 英语发音音频必须优先来自离线校验资源；
- 系统 TTS 只作资源缺失兜底；
- 口语质量影响战斗演出强度，不允许低于最低识别置信度时直接判错。

## 7. 等级、军衔与经济

### 7.1 航员等级

- 等级范围 1–50；
- 累计经验不下降；
- 累计升级经验：

`T(L) = round(60 × (L - 1)^1.55)`

经验来源：

- 新知识首次完成：10 XP；
- 到期复习完成：14 XP；
- 提前复习：4 XP；
- 拼写题乘 1.25；
- 口语题乘 1.30；
- 首次答对乘 1.0；
- 纠错后答对乘 0.65；
- 跳过或设备失败乘 0.2。

同一知识点同一天非到期练习从第 3 次起不再产生 XP。

功能解锁：

- 2 级：武器升级；
- 3 级：宠物舱；
- 5 级：局内战术芯片；
- 8 级：Boss 悬赏；
- 后续等级主要解锁外观、构筑槽和基地陈列。

### 7.2 军衔

- 军衔每 5 级晋升；
- 只提供称号、头像框、基地装饰和入场播报；
- 不提供独立战斗数值；
- 旧军衔作为迁移依据保留。

### 7.3 星域进度

- 一个教材单元拆成 3–5 个短任务；
- 每个任务最多 3 枚星章：
  - 完成任务；
  - 达到目标首次正确率；
  - 完成对应到期复习；
- 下一任务只要求完成前置任务，不强制满星；
- 满星解锁剧情、Boss 外观和收藏。

### 7.4 货币

仅保留两种可消费资源：

- **合金：** 主线和普通任务稳定产出，用于武器升级；
- **星晶：** 到期复习、首次满星和 Boss 首杀产出，用于宠物外观、基地玩具、弹道颜色和终结动画。

星晶不用于武器数值升级、宠物孵化或宠物进化；核心成长只由主线、有效学习和合金驱动。

### 7.5 日常任务

每天固定三个短任务：

1. 完成任意一场；
2. 清理到期复习；
3. 完成一个弱项题型。

全部任务应能在 10–15 分钟内完成。七格航行日志只累积点亮，不因漏一天清空。

## 8. 战斗系统

### 8.1 状态机

战斗状态：

`Entering → QuestionFocus → CommandResolve → Presentation → PhaseCheck → QuestionFocus/BossFinish → Settlement`

每个状态必须有明确进入、退出和超时条件。表现资源缺失或动画回调丢失时，超时后进入下一合法状态，不能卡死存档。

### 8.2 知识装甲与核心生命

Boss 每阶段拥有 3–4 个知识装甲节点：

- 一次有效作答最多破坏一个节点；
- 武器、暴击和宠物不能跳过节点；
- 节点清空后才结算核心生命伤害；
- 高伤害影响演出、奖励和星章，但不减少必须完成的核心题量。

该设计替代当前“按武器估算 Boss 血量并强制保底 1 HP”的隐式规则，使学习门槛对玩家可解释。

### 8.3 伤害

`Damage = BaseDamage × CommandQuality × WeaponMultiplier × MomentumMultiplier × TacticalMultiplier + PetDamage`

约束：

- 动量 0–5 层；
- 每层动量增加 8% 表现伤害；
- 首次答对动量 +1；
- 答错动量 -2，不再全部清零；
- 纠错通过不增加动量；
- 武器 1–10 级总基础伤害差控制在约 35%；
- 伤害不能影响掌握度。

### 8.4 Boss 阶段

每个 Boss 包含：

1. 护盾阶段；
2. 装甲阶段；
3. 核心阶段。

Boss 机制由本局弱项决定。例如：

- 听力弱项：声波护盾，提高听力题权重；
- 拼写弱项：文字裂解，引入字母块变式；
- 阅读弱项：镜像文本，但不改变字号和可读性；
- 口语弱项：回声核心，要求跟读和识别。

机制只能改变题型组合和演出，不能通过干扰视觉、噪声或时间压力降低题目可读性。

### 8.5 失败

- 飞船护盾归零进入“紧急撤离”；
- 保留知识进度和基础奖励；
- 失去本局额外星章；
- 不扣宠物、不损装备、不清航行日志、不强制真实等待；
- 结算页给出 1–2 个具体弱项和再次挑战入口。

### 8.6 Roguelite 强化

每 3 题提供两项本局强化：

- 强化只影响战斗表现、防御、资源转化或武器机制；
- 不修改正确答案和复习调度；
- 每项必须有图标、短标题和一句效果；
- 低年级默认只显示明确的攻击/防御二选一；
- 高年级允许流派构筑；
- 同局最多选择 3 次，避免决策压过学习。

## 9. 武器系统

### 9.1 共同结构

- 1 件主武器；
- 2 个永久模块；
- 等级 1–10；
- 3、7 级各解锁一次二选一模块；
- 5 级解锁专属必杀；
- 10 级解锁终极模块；
- 1、5、10 级更新视觉形态；
- 模块可在基地免费重置。

升级成本：

`UpgradeCost = 80 + 40 × (CurrentLevel - 1)` 合金。

### 9.2 五个武器家族

#### 脉冲炮

- 定位：稳定、新手、防御；
- 每 3 次有效指令生成护盾；
- 纠错时动量损失减半；
- 演出：单发聚能、清晰弹道、稳定冲击。

#### 等离子双炮

- 定位：连击、多段；
- 连续首次答对积累 1–3 枚浮游炮；
- 攻击时多段齐射；
- 动量下降时只损失一枚浮游炮。

#### 烈焰喷射器

- 定位：破甲、持续；
- 首次答对叠加灼烧；
- 阶段结束统一引爆；
- 灼烧只增加核心伤害，不跳过知识节点。

#### 冰霜晶炮

- 定位：防御、控场；
- 每 2 次有效指令冻结 Boss 攻击计数 1 格；
- 同时生成屏障；
- 适合弱项训练和低压力复习。

#### 雷神电弧炮

- 定位：到期复习、爆发；
- 到期复习首次答对产生连锁；
- 连续完成 3 个不同知识点触发天雷处决；
- 不鼓励重复刷同一简单知识点。

### 9.3 获得与外观

- 所有功能武器通过主线蓝图获得；
- 不做随机抽取；
- 星晶只购买外观、弹道颜色和终结动画；
- 高阶武器不能自动答题或跳过学习节点；
- 旧武器 ID 原样迁移。

## 10. 宠物系统

### 10.1 编队

- 一只先锋宠常驻战场；
- 一只支援宠在条件满足时切入；
- 支援宠平时只显示通讯头像；
- 每组双宠组合拥有固定共鸣技；
- 共鸣演出首次完整播放，后续允许跳过。

### 10.2 三只首发宠物

#### 星尘狐

- 定位：侦察、护盾；
- 首次答对标记 Boss 弱点；
- 3 个标记后高速突进并生成护盾；
- 觉醒后增加星尘残影和范围标记。

#### 星云猫

- 定位：动量、幻影；
- 答错时保护 1 层动量；
- 连续首次答对召唤幻影炮击；
- 觉醒后形成三道星云残像。

#### 水晶龙

- 定位：蓄能、终结；
- 有效指令积累晶核；
- 阶段末释放水晶吐息；
- 觉醒后在核心阶段追加俯冲终结。

### 10.3 羁绊与进化

- 羁绊等级 1–20；
- 每次有效作答 +1 羁绊 XP；
- 到期复习首次答对额外 +1；
- 每日首次基地互动 +3；
- 5 级进入成长形态并解锁主动技；
- 12 级完成专属学习任务后进化；
- 20 级星辉觉醒并解锁终极共鸣；
- 宠物蛋来自主线里程碑；
- 完成 3 次有效学习后孵化；
- 无饥饿、死亡、离家、降级或忘记照顾惩罚；
- 皮肤和玩具不改变能力。

## 11. UI 与题型布局

### 11.1 屏幕方向

首发全局横屏。基地、星图、战斗和报告均保持横屏，避免进入战斗时旋转设备。

### 11.2 全屏专注答题

答题阶段覆盖战场主要交互：

- 战场暂停或使用低对比度静态背景；
- 题干、音频、选项和输入占据完整安全区；
- 退出、重复播放和辅助按钮位置固定；
- 进入结果前禁止播放攻击音效。

### 11.3 题型布局

选择、阅读、听力：

- 默认 2×2；
- 长句使用 1×4；
- 音频播放键与题干固定在顶部；
- 选项不随文字长度缩小到不可读。

拼写：

- 题干顶部；
- 答案槽居中；
- 字母池位于底部两行；
- 低年级使用拖放或点击字母块；
- 高年级可启用完整键盘；
- 默认不调用系统软键盘。

口语：

- 目标句、播放示范、录音波形、识别文本、重录按钮同时可见；
- 默认允许一次重试；
- 录音阶段禁止 Boss 音效；
- 无识别能力时进入辅助模式。

## 12. 音频、动画、特效与触觉

### 12.1 音频总线

逻辑总线：

- Master；
- BGM；
- Ambience；
- SFX；
- Voice；
- UI。

设置分别保存音量和开关。

### 12.2 动态音乐

音乐状态：

- 基地；
- 普通战斗；
- Boss 阶段；
- 核心暴露；
- 胜利。

状态切换使用 0.4–0.8 秒交叉淡化。英语题目播放时，BGM 和环境音降低约 12 dB，战斗 SFX 暂停。

### 12.3 教学语音

- 核心单词和句型使用预生成的离线音频；每个单元人工试听不少于 10% 且不少于 5 条，音量、截断或发音异常项必须全量复检；
- 音频通过 `contentId` 引用；
- 系统 TTS 只作缺失兜底；
- 中文解说只用于入场、Boss 阶段和重要里程碑；
- 中文解说不得与英语教学音频重叠。

### 12.4 音效

每件武器至少包含：

- 蓄能；
- 发射；
- 飞行；
- 命中；
- 特殊触发。

同类短音效设置 80–120 ms 防重复窗口，并提供 3 个轻微变体。

### 12.5 无 Spine 动画管线

普通角色：

1. AI 生成概念；
2. 人工统一轮廓与配色；
3. 拆分头、躯干、四肢、武器和特效层；
4. 使用 Cocos 分层 Node 层级绑定；
5. 使用 Animation Clip 驱动位移、旋转、缩放和 SpriteFrame 替换；
6. Shader、ParticleSystem2D 和拖尾提供发光、残影与能量流动。

每个单位统一支持：

- idle；
- anticipate；
- attack；
- hit；
- break；
- skill；
- victory。

形变明显的 Boss 必杀可使用短 Sprite Sheet，但不得把所有动作制作成逐帧动画。

### 12.6 表现事件

一次攻击拆为：

`prepare → launch → impact → recover`

逻辑伤害在 `impact` 落地。若动画资源缺失或回调未触发，超时后仍结算并继续。

命中反馈：

- 轻命中停顿约 33 ms；
- 重命中约 67 ms；
- 终结约 100 ms；
- 停顿不得冻结教学语音；
- 普通攻击禁止连续全屏闪白。

### 12.7 触觉

通过 Native Bridge：

- iOS：轻、中、重反馈；
- Android：VibrationEffect 对应轻、中、重模式；
- 设置中可独立关闭；
- 不支持设备静默降级。

### 12.8 性能预算

- 中高端目标 60 FPS；
- 低端保底 30 FPS；
- 高档活动粒子不超过约 400；
- 低档活动粒子不超过约 120；
- 同时播放声音不超过约 16；
- Boss 与关卡专属资源离开战斗后释放；
- 每个特效提供高、中、低三档；
- “减少动态效果”关闭强震屏、快速闪光和背景视差。

## 13. 软件架构

### 13.1 分层

- `content/`：教材、语言点、题型模板、音频清单和校验；
- `learning/`：FSRS、掌握度、错题重排和出题策略；
- `battle/`：状态机、Boss、伤害和强化；
- `progression/`：等级、星域、货币、武器和宠物；
- `presentation/`：场景、动画、粒子、镜头和音频；
- `platform/`：文件、分享、触觉和原生 TTS；
- `storage/`：版本、原子写入、备份和迁移；
- `ui/`：Profile、Base、StarMap、Battle、Report。

### 13.2 依赖方向

`presentation/ui → battle/progression → learning/content`

`platform/storage` 通过接口注入。

禁止：

- 学习领域引用 Cocos Node；
- UI 直接写存档；
- 武器或宠物修改答案判定；
- 表现动画持有唯一业务状态。

### 13.3 场景

- Boot：配置校验、存档恢复、迁移；
- Profile：孩子、教材、年级；
- Base：任务、养成、设置；
- StarMap：教材单元和路线；
- Battle：题目与战斗；
- Report：学情、历史和导入导出。

### 13.4 事件

至少定义：

- `QuestionPresented`；
- `QuestionAnswered`；
- `CorrectionCompleted`；
- `MasteryUpdated`；
- `CommandResolved`；
- `AttackResolved`；
- `BossPhaseChanged`；
- `PetSkillTriggered`；
- `BattleFinished`；
- `SaveCommitted`。

事件载荷使用明确 TypeScript 类型。事件总线不得使用任意字符串和无类型对象。

## 14. 内容架构

### 14.1 分包

现有 `data.js` 与 `data-kouyu.js` 转为：

`教材 / 年级 / 学期 / 单元.json`

战斗只加载当前单元及通用题型资源。

### 14.2 校验

构建和启动开发模式至少检查：

- `contentId` 唯一；
- 英文和中文非空；
- 题型引用有效；
- 音频引用存在或明确标记 TTS 兜底；
- 干扰项数量充足；
- 正确答案不出现在干扰项中；
- 单元和教材引用有效；
- 生成文件不可手工覆盖源数据。

### 14.3 资产规范

每个外部资产记录：

- 来源；
- 作者；
- 许可证；
- 是否允许修改和分发；
- 原始文件；
- 加工步骤；
- App 内资源路径。

AI 资产记录提示词版本、参考风格和人工修订责任人。

  `1/3/5/8/12/16/20/25/30/40/50`；

## 16. 错误处理

### 16.1 配置错误

- 开发构建直接失败并报告文件和字段；
- 发布构建跳过损坏单元并在诊断页记录；
- 核心通用配置损坏则阻止进入主界面。

### 16.2 资源错误

- 缺失角色资源使用明确的通用占位资源；
- 缺失音效静默降级；
- 缺失教学音频尝试系统 TTS；
- 动画回调超时后继续逻辑；
- 所有降级都写入本地诊断日志。

### 16.3 平台错误

- 触觉不支持时静默关闭；
- 文件选择取消不视为错误；
- 导出失败保留本地文件并提供重试；
- 录音权限拒绝时切换辅助口语模式；
- 存储空间不足时停止新写入并提示先导出。
